🖥️ Apps Script 접속 방법

Apps Script = 코드 및 트리거(각각의 함수가 자동 실행되는 주기를 설정하는 것)를 관리하는 구글 도구입니다.

STEP 2더힘이엔씨 일일업무보고 ㄹㅇㄹㅇ 스프레드시트 열기

📁 파일 구성 — 실행 순서대로
ℹ️
함수(Function)란?
함수는 특정 작업을 수행하는 코드 묶음입니다. 예를 들어 generateBriefing()은 "브리핑을 생성하라"는 작업 전체가 담긴 코드 묶음입니다.

Apps Script에서 각 파일을 열면 상단 디버그 버튼 옆 드롭다운에 해당 파일의 실행 함수 이름이 표시됩니다. 각 파일마다 실행되는 함수는 하나로 설정돼있으며, 이 함수 이름이 트리거와 연결됩니다.

⚠️ 함수 이름은 왠만하면 수정하지 않는 것을 강력히 권장합니다. 함수 이름을 바꾸면 트리거와 연결이 끊겨 자동화 전체가 중단됩니다.
1
doPost()
직원보고수신(힘클로드, 제형).gs

이 함수가 하는 일: 직원이 힘클로드 HTML 파일에서 업무 내용을 입력하고 제출 버튼을 누르면 자동으로 이 함수가 호출됩니다. Claude API를 이용해 입력 내용을 JSON 형식으로 변환하고, luri7305@gmail.com으로 [일일보고] 메일을 발송합니다.

실행 방식: 트리거 없음. 힘클로드 HTML 파일이 이 함수가 있는 웹앱 URL로 데이터를 전송할 때만 실행됩니다. (웹앱 배포 방식)

실행 방식: 트리거로 5분마다 자동 실행. 삼촌 계정 소유 트리거 필수.

당부 메시지

이 시스템을 인수받아 관리하게 되신 분께.

함수 및 트리거는 여러 개 만들지 마세요. 여러 개 만들면 뭐가 뭔지 헷갈리고 서로 충돌해서 오류가 백퍼센트 발생하게 됩니다.

하다보면 오류도 분명 생길 것이고, 코드를 수정하다보면 상당히 열받는 일들이 많이 생길 것입니다.

제가 꼭 당부드리고 싶은 것은, 코드를 수정하기 전에는 항상 꼭 백업 파일을 저장해두세요. 수정한 파일이 실제로 작동하는지 테스트하려면 Apps Script 상단의 실행 버튼 왼쪽에 있는 Drive 프로젝트 저장 버튼을 꼭 먼저 누르고 테스트해야 합니다. 저장하지 않으면 수정 전의 예전 파일로 계속 작동하게 돼서 토큰이 낭비됩니다.

그리고 오류가 계속 떠서 뭐가 문제인지 확인하고 싶으시면, 실행 후 하단에 뜨는 로그와 왼쪽에 있는 실행 탭을 잘 확인해보시길 바랍니다. 거기서 대부분의 원인을 찾을 수 있습니다.

레벨별 보고서 가이드라인 수정 방법
수정 가능

현재 L1~L4 보고서는 큐 생성 후 브리핑 생성하는 코드 파일 안의 두 곳의 텍스트를 기준으로 Claude AI가 작성합니다. 보고서의 형식, 포함 내용, 강조 사항을 바꾸고 싶으면 이 두 부분만 수정하면 됩니다.

📍 수정 위치: Apps Script → 큐 생성 후 브리핑 생성하는 코드 파일

① 수정 위치 1 — levelDesc (각 레벨 보고서가 어떤 내용인지 AI에게 알려주는 텍스트)

이 변수가 각 레벨 보고서의 목적과 대상을 정의합니다. AI는 이 텍스트를 읽고 "이 보고서는 이런 내용을 담아야 한다"고 이해하고 작성합니다. 텍스트를 더 구체적으로 바꿀수록 보고서 품질이 올라갑니다.
